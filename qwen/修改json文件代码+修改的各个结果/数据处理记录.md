# 数据处理记录

## 文件变更历史

### 原始文件
- `my_annotations.json` - 原始标注数据

### 处理流程

#### 1. 添加九宫格描述
**使用脚本**: `add_grid_description.py`
**输入文件**: `my_annotations.json`
**输出文件**: `my_annotations_with_box.json`
**处理内容**:
- 读取 YOLO 格式的边界框标注（`labels/` 目录）
- 计算边界框覆盖的九宫格位置
- 在助手回复中添加九宫格描述：`"火焰主要占据了图片九宫格的XXX板块。"`
- 插入位置：`"是的，这是一张火灾图像。\n\n"` 之后

**九宫格定义**:
```
左上  中上  右上
左中   中   右中
左下  中下  右下
```

**配置参数**:
```python
IMAGES_DIR = "images"
LABELS_DIR = "labels"
JSON_FILE = "my_annotations.json"
OUTPUT_FILE = "my_annotations_with_box.json"
```

---

#### 2. 更新用户提问
**使用脚本**: `update_user_question.py`
**函数**: `update_user_questions()`
**输入文件**: `my_annotations_with_box.json`
**输出文件**: `my_annotations_updated.json`
**处理内容**:
- 将用户提问从：
  `"请先判断是否为火灾图像，再详细描述这张图片的内容。"`
- 改为：
  `"请先判断是否为火灾图像，再说明火焰占据图片九宫格的哪几个版块。"`

---

#### 3. 删除详细描述
**使用脚本**: `update_user_question.py`
**函数**: `remove_detailed_description()`
**输入文件**: `my_annotations_updated.json`
**输出文件**: `my_annotations_updated_final.json`
**处理内容**:
- 删除九宫格板块描述后面的详细图片描述
- 保留格式：
  ```
  是的，这是一张火灾图像。

  火焰主要占据了图片九宫格的XXX板块。
  ```
- 删除：`"板块。"` 之后的所有内容

---

#### 4. 更新图片路径
**使用脚本**: `update_user_question.py`
**函数**: `update_image_paths(image_prefix="images/")`
**输入文件**: `my_annotations_updated_final.json`
**输出文件**: `my_annotations_updated_final.json`（覆盖）
**处理内容**:
- 将图片路径从单纯的文件名改为相对路径
- 例如：`00000.jpg` → `images/00000.jpg`
- 方便 `train.py` 直接读取图片

**路径验证命令**:
```bash
cd /d "d:\VLM\fire-yolo-vlm\txt-box" && python -c "import os; print(os.path.exists('images/00000.jpg'))"
```

---

#### 5. 严格指令约束（更新用户提问）
**使用脚本**: `update_user_question.py`
**函数**: `update_user_questions()`
**输入文件**: `my_annotations_updated_final.json`
**输出文件**: `my_annotations_updated_final_prompt.json`
**处理内容**:
- 将用户提问改为：
`"请先判断是否为火灾图像。如果是，请说明火焰占据图片九宫格的哪几个版块；若不是，则直接回答不是火灾图像即可，无需分析版块。<|vision_start|>images/00000.jpg<|vision_end|>"`
